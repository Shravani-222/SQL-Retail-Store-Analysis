# 🛒 SQL Retail Store Analysis

## 📌 Project Overview
This project focuses on analyzing a retail store database using SQL. It includes multiple real-world business queries related to customers, products, orders, inventory, and shipping.

The goal is to extract meaningful insights from structured data using SQL queries.

---

## 🗂️ Dataset Tables Used
- ONLINE_CUSTOMER
- PRODUCT
- PRODUCT_CLASS
- ORDER_HEADER
- ORDER_ITEMS
- ADDRESS
- SHIPPER
- CARTON

---

## 🔍 Key SQL Concepts Used
- Joins (INNER JOIN, LEFT JOIN)
- Subqueries
- Aggregate Functions (SUM, COUNT)
- GROUP BY & HAVING
- CASE Statements
- Filtering & Sorting
- Data Transformation

---

## 📊 Business Problems Solved

### 1. Customer Segmentation
- Categorized customers into A, B, C based on account creation year.

### 2. Unsold Products Analysis
- Identified products not sold.
- Calculated inventory value and applied dynamic discounts.

### 3. Inventory Analysis by Product Class
- Found high-value inventory product classes.

### 4. Customer Behavior Analysis
- Identified customers who cancelled all their orders.

### 5. Shipping Insights
- Analyzed DHL shipping performance by city.

### 6. Cash Payment Analysis
- Found customers paying via cash with specific name patterns.

### 7. Logistics Optimization
- Determined largest order fitting in a specific carton.

### 8. Inventory Status Classification
- Categorized inventory as Low, Medium, or Sufficient.

### 9. Product Bundling Insights
- Found products frequently sold with a specific product.

### 10. Order Filtering
- Extracted even order IDs with specific shipping conditions.
